# OweManager v2 要件定義書

# 1. プロダクト概要

OweManagerは、友人間で発生する少額の貸し借りを記録・管理するためのWebアプリケーションである。

v1では「借りたお金」の管理のみを対象としていたが、v2では「貸したお金」も管理対象に含める。

また、v1を友人に利用してもらった結果、

* LINEやメモアプリでも代替できる
* 記録するまでのハードルが高い

という課題が見つかったため、v2では管理機能の追加ではなく「記録導線の改善」を重視する。

---

# 2. 背景

## 現状

* 友人と食事に行った際に代金を立て替えてもらうことが多い
* 現金をあまり持ち歩かない
* 「今度返す」と約束することが多い
* バイト代が入るまで待ってほしいと言われることもある
* 誰にいくら借りたか忘れる
* 誰にいくら貸したか忘れる
* 返済済みかどうか曖昧になる

---

## 現在の管理方法

### LINE

借りた相手とのトークで管理

課題

* メッセージに埋もれる
* 一覧で確認できない
* 貸し借りの全体像を把握しづらい

---

### メモアプリ

名前と金額を記録

課題

* 返済済みかどうか分かりづらい
* 消し忘れが発生する
* 貸したお金と借りたお金を整理しにくい

---

# 3. ターゲット

大学生

特に、

* 食事
* 飲み会
* 旅行
* 立て替え

などで友人間の貸し借りが頻繁に発生する人

---

# 4. 解決したい課題

貸し借りが発生した瞬間に記録するハードルが高く、その結果として管理が曖昧になること。

---

# 5. MVP仮説

ユーザーが本当に困っているのは管理機能不足ではなく、

「記録するまでの手間」

である。

そのため、

* 入力項目を最小限にする
* スマホ利用を前提とする
* ホーム画面から素早く起動できる

ことを重視する。

---

# 6. 検証したい仮説

* PWAによって記録導線が改善される
* 貸したお金管理にも需要がある
* メモアプリより継続利用される
* 入力項目を最小化することで記録率が向上する

---

# 12. DB設計

## users

| カラム名       | 型         |
| ---------- | --------- |
| id         | UUID      |
| email      | VARCHAR   |
| name       | VARCHAR   |
| created_at | TIMESTAMP |
| updated_at | TIMESTAMP |

補足

将来的なプロフィール編集機能追加に備えて updated_at を保持する。

---

## debts

| カラム名         | 型         |
| ------------ | --------- |
| id           | UUID      |
| user_id      | UUID      |
| partner_name | VARCHAR   |
| amount       | INTEGER   |
| type         | ENUM      |
| status       | ENUM      |
| created_at   | TIMESTAMP |
| updated_at   | TIMESTAMP |

補足

updated_at は以下の場合に更新する。

* 編集時
* 返済済み変更時
* 未返済への戻し時

---

## partner_nameについて

MVPでは文字列として保持する。

理由

* 入力速度を優先するため
* 実装をシンプルにするため

将来的に、

* 表記ゆれ
* 相手候補表示
* 相手ごとの集計

などが課題となった場合は partners テーブルの追加を検討する。

# 13. API

## 一覧取得

GET /api/debts

### クエリパラメータ

| パラメータ  | 値               |
| ------ | --------------- |
| type   | borrowed / lent |
| status | unpaid / paid   |

例

GET /api/debts?type=borrowed&status=unpaid

---

## 新規登録

POST /api/debts

---

## 詳細取得

GET /api/debts/:id

---

## 更新

PATCH /api/debts/:id

---

## 削除

DELETE /api/debts/:id

---

# 14. 認可設計

すべての貸し借りデータは user_id に紐づく。

取得・更新・削除時は、

WHERE id = :id
AND user_id = :currentUserId

を条件とする。
